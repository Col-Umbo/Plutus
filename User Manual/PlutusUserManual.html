Introduction & Disclaimer

        This user manual is an introduction and tour of the application and all its features. This document is intended to show users how to use the various features of the application, things to watch out for, and a recommendation on how to use Plutus. Due to the designed simplicity of the application, this user manual only uses a compilation of pictures and a description to show the different pages and their accompanying features and functions which are organized in sections. There shouldn’t be any learning curve to using the application, but it may be helpful to look through this document before use of the application.

It is important to note that the order in which we are showing how to use the application isn’t mandatory. You can use different functions of the application separately or in any order or combination. For example, you can go ahead and start logging transactions in the Transactions page right away! You don’t have to perform any order of operations before using the Transactions page, or any other page for that matter. With that said, here is how we recommend using the application for the most optimal and seamless experience:

  1. Categories Page

Add spending and income categories here in the Categories page (recommended to do first). To add a category, first navigate to the Categories page with the navigation bar at the bottom. Then click the “+” button at the top, right corner. From there, you can name the category and choose a unique color for that category. Then click “Add” and the new category will be added.

Once some categories are listed, you can sort through them by expense or income categories. In the filter bar below the Category Totals section, click “Income” to show only income categories and click “Expenses” to show only expense categories. To show all categories, click “All”. The Category Totals card above shows how many of each category exists, and you can make changes to the categories in the list below the filter card.

You can edit categories as needed, or you can delete the category if you feel that it’s no longer relevant. Note that deleting a category may affect other parts of the program; if you delete a category, it will also remove any accompanying transactions and the budget allocation of that category (see the budget page section next). Before deleting a category, make sure to make changes in the Transactions page before doing so (also see the Transactions section)

Adding known spending and income categories first allows you to easily plan and set a budget, as well as properly categorizing transactions. That’s why we recommend stopping here first before using the rest of the features in Plutus.

  1. Budget Page

Set a max budget, and allocate funds from the budget to each category. Note that there will be no dealing with real-world money within the application. This is used to plan out your finances. Allocating funds to each category helps you visualize setting a spending limit on certain things, for example, you want to set a budget of $40 for subscriptions, so you allocate $40 of the max budget to the Subscriptions category. To get to the Budget page, simply click the Budget button on the navigation bar.

The list of status bars in the “Allocated Categories” section is connected to the Transactions page. It automatically updates when you add a transaction. The status bar will fill as you add a transaction within the respective category. If you overspend in a particular category, the status bar will fill completely and turn red, letting you know that you went over budget.

You can also delete category budget allocations if you feel that you do not need it anymore. If you happen to delete a category, the budget allocation assigned to that category will also be deleted.

  1. Transactions Page

The Transactions page is where you can log and keep up with all of your transactions. Click the Transactions button on the navigation bar to navigate there. You can either log expenses or income on this page. To do so, click the “+” button at the top right hand corner of the “Transactions Totals” card. From there, a modal will appear where you can set the type of transaction (income or expense), amount, date, category, name, and whether it will repeat in the future. To add, click “Add”, and the new transaction will appear at the list below.

Like the Categories page, you can filter transactions by type (income or expense). There is no way, as of now, to filter by category, but you can search for a transaction by category or name using the search bar. To filter a transaction by expense, click the “Expense” button in the filter bar; to filter by income, click the “Income” button. Your total amount of expenses and total amount of income will be shown in the “Transactions Totals” card above.

You can make changes to each transaction in the list at the bottom. Click the trashcan button to delete the transaction, or click “Edit” to open the modal back up to make changes to the transaction.

Transactions are connected to the Summary and Budget pages (see the Summary page section further down). Once again, the transactions are connected to the category budget allocations bar in the Budget page. You can add transactions here and go to the Budget page to keep track of a particular category.

  1. Goals Page

The Goals page is an off-shoot feature that doesn’t make any changes to the Budget or Transactions pages (affecting your logged finances), but adding goals can give insights into how it will affect your current finances. Navigate to the Goals page by clicking the Goals button on the navigation bar.

To add a goal, simply fill out the fields in the “Add What-If Scenario” card. Choose the type (Income or Expense), the amount the affecting scenario or goal will need, the category it should be associated with, the name, and an optional description. Then click “Add Scenario” to add the scenario, or “Clear All” to clear all inputs.

        Once a scenario is added, it will appear in the Scenario List below. On the side, there will be a Projection Summary that shows how all added scenarios will impact your current finances. Below that shows how each scenario will affect each category and its allocated budget.

        To remove a scenario from the Scenario List, simply click on the trashcan icon on the right side of the corresponding scenario list item. Doing so will adjust the information displayed on impacts to current finances and budget.

  1. Summary Page

        The Summary page gathers all logged information and shows an overview, or summary if you will, of all your logged transactions. Once you first open up the application, you automatically start at the Summary page. However, like all other pages, you can always navigate to the Summary page by clicking the Summary button in the navigation bar.

        The Summary page mostly includes visual information about everything you have logged so far this month. The Summary page should reset every month to show current, relevant information for the current month. This page includes several cards that shows various data. These cards include:

  • This Month’s Snapshot
  • Total Spending
  • Budget Overview
  • Save and Import Data

        This Month’s Snapshot shows the current month’s overview, such as how much income you’ve accumulated, how much you have spent, net cash flow (income minus expenses), and the remaining balance of your overall budget that you set on the Budget page.

        The Total Spending card includes a donut chart that shows how much in total you have spent this month with the added information on how much you spent in each category. The donut chart displays all the categories with logged transactions and can give insight into how much you spend in a particular category and where you may need to cut back. Off to the side, there is a list that shows your top spending categories organizes each category with a logged transaction from the most spent to the least spent category. In the example below, I have spent the most in the Bills category as shown on the graph, and in more detail in the accompanying list.

        The next card is the Budget Overview, where you can get information on your budget health and recent activity. The Budget Health sub-card shows how much of your budget was used, how many days left in the month, and how much you are projected to spend. Next to that card is the Recent Activity where you can view your recent transactions.

        

Below the Budget Overview card is the Save and Import Financial Data card where you can save all of the transactions and categories you logged, or you can import previously saved data and bank statements (.csv file only). When you want to save data, click the “Export” button. From there, you can name the file and save it as a .csv file, and it will be stored in downloads, or in whichever folder the Plutus application is stored on in your computer.

        If you want to import data (previously saved data or bank statement from .csv file) then click “Import” and a modal will appear. In the modal, you can drag and drop the file into the box, or you can browse your files. If you wish to cancel, then click “Cancel”. Once you choose the .csv file to import, click open, and all financial information will be populated in the Transactions and Categories pages. The Budget page will not be affected by this. You should get a persisting confirmation message below the card that confirms that the import was successful.

  1. Password Lock

        If you want to secure the application, we have included a feature to do so! Even though this application is completely offline, and no one will be able to access your information remotely through the application itself, you can still prevent someone from getting your financial data physically. To do this, you can utilize the lock feature. You can be on any page in the application to enable this feature. To enable, click the lock icon at the top right of the page on the header bar. When clicked, a modal will appear and will ask you to enter and reenter a password to password lock the application on startup.

        There are no restrictions on password length or specific types of characters for the password. In the input boxes, type your password then retype your password to ensure that you can remember it or that you have typed in the correct characters, then click “Submit”. If the passwords don’t match between the Set and Reenter input boxes, then you will not be able to set the password, and an error message will display. If you did not mean to click on the password lock modal, simply click “Cancel” or click away from the modal.

        Once the password is set, it doesn’t immediately seem like anything has changed. However, you can confirm that the password was set by clicking the Password Lock button again.

        If a password lock is enabled, then the password lock button will display a “Disable Password Lock” modal. From there, you can change the password, or you can disable the password lock. BE CAREFUL to not forget your password. If you forget your password, there is no recovery option. To disable the password lock, simply type in your password and click “Disable Lock”. There will be an error message if you type in the incorrect password, and you will not be able to disable the lock. However, if you want to change the password, then click “Change Password”, and the modal will display a different set of inputs.

        Here, you can type in your old password then enter and reenter the new password. If you typed in the old password incorrectly, or your new password entries don’t match, then you will receive an error message and you will not be able to change the password.

        Now, when you close out of the application and reopen it, you will be greeted with the password lock modal that blocks the main content of the screen. You will not be able to click away from the modal, and only entering the correct password will let you through.

        Enter the correct password then click submit, and you will be directed to the Summary page as normal, and you will be able to use the application. Also note that when you enable the password lock, the accompanying database for the application will also be encrypted so that no one can see its contents in plain text outside of the application.

  1. Light / Dark Mode

        If you’re tired of the look and feel of dark mode, you can change it to a light mode! Light mode changes the look and feel of the application to a lighter, cleaner look. Unfortunately, there are no further options to change between any other theme other than light or dark mode, but the option between light and dark is there to switch to your liking. To switch between light and dark mode, click “Light” at the top in the header bar to switch to light mode, and then click “Dark” to switch back to dark mode.
